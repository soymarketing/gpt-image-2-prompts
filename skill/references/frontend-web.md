# Frontend / Web Design

Use this reference when the user wants prompts for landing pages, SaaS hero sections, UI mockups, dashboard shots, app screens, design explorations, homepage concepts, feature sections, ecommerce layouts, or web design inspiration images.

## What matters most

Prioritize:
- layout clarity
- hierarchy
- realistic product/UI framing
- clean copy space
- controllable visual system
- believable modern design taste
- device and screen context only when it helps

## Prompt structure for frontend/web design

```text
Create a [landing page hero / homepage concept / dashboard UI / mobile app screen / feature section visual] for [product or brand].
Audience: [who it is for]
Goal: [what the design should communicate]
Core product/theme: [AI design tool, fintech dashboard, wellness app, ecommerce brand, etc.]
Layout: [desktop / mobile / split layout / bento grid / dashboard / single-column]
Main sections: [hero, stats, features, testimonials, pricing preview, etc.]
Visual style: [minimal, premium SaaS, bold startup, editorial, futuristic, calm wellness]
Color system: [palette]
UI details: [cards, charts, nav, CTA buttons, screenshots, gradients, glassmorphism, etc.]
Composition: [centered hero, left copy + right product shot, clean whitespace, screen mockup angle]
Text to include: "[EXACT TEXT]" if necessary
Constraints: no extra text, no clutter, clean hierarchy, presentation-ready, modern design taste
```

## Good language for frontend/web prompts

Use phrases like:
- `landing page hero`
- `premium SaaS homepage`
- `modern dashboard UI`
- `bento grid layout`
- `clean hierarchy`
- `conversion-focused`
- `high-end product design mockup`
- `minimal whitespace-rich interface`
- `design system consistency`
- `clear primary CTA`

## Common prompt patterns

### 1. SaaS landing page hero

```text
Create a premium SaaS landing page hero concept for an AI automation platform.
Audience: startup operators and marketers.
Goal: communicate speed, intelligence, and trust.
Core product/theme: workflow automation with AI agents.
Layout: desktop website hero with left copy column and right product mockup.
Main sections: hero only, with headline area, short supporting copy, CTA zone, and product screenshot.
Visual style: modern premium SaaS, clean and sharp.
Color system: off-white background, charcoal text, subtle blue gradients.
UI details: polished dashboard cards, simple charts, refined CTA button, tasteful shadows.
Composition: strong whitespace, product mockup on the right, clean visual hierarchy.
Text to include: "Automate the work that slows you down".
Constraints: no extra text, no clutter, modern product design taste, conversion-focused.
```

### 2. Dashboard concept

```text
Create a modern analytics dashboard UI for an ecommerce operations tool.
Audience: store operators and growth teams.
Goal: make performance, inventory, and alerts easy to scan.
Core product/theme: ecommerce command center.
Layout: desktop dashboard with top navigation, KPI cards, charts, and activity feed.
Main sections: KPI summary row, charts area, alerts panel, recent activity list.
Visual style: crisp modern B2B SaaS design.
Color system: white and slate base with blue and green status accents.
UI details: line charts, metric cards, compact table rows, clean icons.
Composition: flat front-facing UI presentation, balanced spacing, clear hierarchy.
Constraints: no extra text, no noisy decoration, clean spacing, presentation-ready.
```

### 3. Ecommerce homepage visual direction

```text
Create a premium ecommerce homepage concept for a modern skincare brand.
Audience: design-conscious online shoppers.
Goal: communicate luxury, softness, and trust.
Core product/theme: minimalist skincare.
Layout: desktop homepage with hero section, product highlights, ingredients strip, and testimonial preview.
Main sections: large hero image, featured products, benefits row, social proof.
Visual style: luxury minimal editorial web design.
Color system: warm neutrals, soft cream, muted sage, dark taupe typography.
UI details: generous spacing, elegant buttons, premium product cards, subtle texture.
Composition: calm high-end layout with strong visual hierarchy.
Constraints: no extra text, no clutter, no cheap ecommerce tropes, polished premium design.
```

## Revision prompts for frontend/web work

- `Keep the layout, make it feel more premium and less template-like.`
- `Preserve the hierarchy, add more whitespace and cleaner spacing.`
- `Make the UI more conversion-focused and less decorative.`
- `Change only the visual style from startup-bold to luxury-editorial.`
- `Keep the structure, make the dashboard easier to scan at a glance.`
- `Preserve the design system, improve CTA prominence.`

## Failure fixes

### Looks like a Dribbble cliché
Add:
- clearer product purpose
- more realistic UI blocks
- stronger hierarchy
- fewer decorative gradients
- `conversion-focused`
- `presentation-ready`

### Too pretty, not usable
Add:
- explicit sections
- audience
- business goal
- `clear navigation and CTA hierarchy`
- `easy to scan`

### Messy layout
Add:
- `clean grid`
- `strong whitespace`
- `clear section breaks`
- `limited component variety`

### Weak product feel
Add:
- what the product actually does
- specific UI modules
- realistic charts/cards/content types
